# Asset Brief — MVP Asset Pack V1

Standalone production asset candidates for the Crash Site MVP, authored in Blender 4.0.2 from the approved MVP concept sheet (workbench / beacon inactive / beacon active / four resources). Generated by `tools/blender/create_mvp_asset_pack_v1.py`; visual-only, collisionless, not integrated into any production scene.

## Shared contract

- Scale: 1 Blender/Godot unit = 1 meter, Z-up authoring, Y-up GLB export.
- Origin: world zero at ground contact for every asset.
- Palette: beige hull `TC_MVP_HullBeige`, dark metal `TC_MVP_DarkMetal`, orange paint/hologram accents, purple alien emissives — matching `docs/art/titancraft-visual-identity.md` and the concept sheet.
- Materials: Principled BSDF node materials only; emissives use Emission Strength (exported as `KHR_materials_emissive_strength`).
- Collision policy: `none` on all meshes (`titancraft_collision` custom property).
- Export: GLB via `tools/blender/export_asset.py` to `assets/Production/Generated/MVP_Pack_V1/`.
- Review: 3 auto-framed PNGs per asset via `tools/blender/render_mvp_asset_review.py` (Cycles CPU, headless-safe).
- Binary delivery: `.blend`/`.glb` are regenerated deterministically by CI or locally; binaries are not committed.
