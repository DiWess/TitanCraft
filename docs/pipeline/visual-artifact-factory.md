# TitanCraft Visual Artifact Factory

## Purpose

The TitanCraft Visual Artifact Factory is a GitHub Actions workflow that makes visual review reproducible in CI. It generates downloadable PNG review artifacts for standalone Blender assets and allowlisted Godot scene capture scripts without committing generated PNGs.

The workflow is review infrastructure only. Generated screenshots and manifests are evidence for human review; artifact generation does not equal visual approval.

## Workflow

- Workflow path: `.github/workflows/visual-artifact-factory.yml`
- Workflow name: `TitanCraft Visual Artifact Factory`
- Manual trigger: `workflow_dispatch`
- Pull request trigger: changes under Blender tools/assets, visual review tools, art/pipeline docs, visual review scenes, `scenes/Main/**`, or the workflow file itself.

## Manual Run

1. Open GitHub Actions.
2. Select **TitanCraft Visual Artifact Factory**.
3. Run the workflow from the target branch.
4. Download the artifact bundles from the completed workflow run.

### `godot-scene-review-bundle`

Expected contents include:

- Every PNG produced by allowlisted Godot visual review capture scripts under `artifacts/visual-review/**`.
- Capture logs under `artifacts/visual-review/logs/`.
- `artifacts/visual-review/visual_artifact_factory_scene_manifest.json`.
- Any contact sheets, runtime contract reports, or visual review manifests generated by the existing capture scripts.

Missing allowlisted scripts are recorded as skipped in the scene manifest. Existing allowlisted scripts must produce at least one PNG or the job fails.

### `visual-artifact-factory-report`

Expected contents include:

- `artifacts/visual-artifact-factory/visual_artifact_factory_manifest.json`
- `artifacts/visual-artifact-factory/generated_file_list.txt`
- `artifacts/visual-artifact-factory/generated_sha256.txt`
- `artifacts/visual-artifact-factory/workflow_summary.md`
- Blender and Godot command logs

## Local Commands

Blender asset review commands can be run locally when Blender is installed:

```bash
blender --background --python tools/blender/create_test_crate.py
python3 tools/blender/validate_blender_asset.py assets/Source/Blender/_templates/TC_TestCrate.blend
blender --background --python tools/blender/export_asset.py -- assets/Source/Blender/_templates/TC_TestCrate.blend assets/Production/Generated/Test/TC_TestCrate.glb
blender --background --python tools/blender/render_asset_turntable.py -- assets/Source/Blender/_templates/TC_TestCrate.blend artifacts/asset-review/test-crate/neutral_turntable.png neutral
blender --background --python tools/blender/render_asset_turntable.py -- assets/Source/Blender/_templates/TC_TestCrate.blend artifacts/asset-review/test-crate/material_turntable.png material
blender --background --python tools/blender/create_heavy_crash_hull_v1.py
python3 tools/blender/validate_blender_asset.py assets/Source/Blender/Production/TC_HeavyCrashHull_V1.blend
blender --background --python tools/blender/export_asset.py -- assets/Source/Blender/Production/TC_HeavyCrashHull_V1.blend assets/Production/Generated/CrashWreck/TC_HeavyCrashHull_V1.glb
blender --background --python tools/blender/render_heavy_crash_hull_v1_reviews.py
python3 tools/blender/build_asset_manifest.py
```

For production-cockpit verification without local Blender artifacts, use the safe manifest check so local validation does not overwrite the checked-in manifest:

```bash
python3 tools/blender/build_asset_manifest.py --check
```

Godot scene review commands can be run locally when Godot and Xvfb are installed:

```bash
godot --headless --path . --import
python3 tools/visual_review/run_visual_artifact_factory.py
python3 tools/visual_review/build_visual_artifact_factory_manifest.py
```

## Registration Rules for Future Review Outputs

- Add new Blender review outputs through explicit tool commands in `.github/workflows/visual-artifact-factory.yml`.
- Add new Godot scene captures to the allowlist in `tools/visual_review/run_visual_artifact_factory.py`; do not execute arbitrary filesystem matches.
- Write logs for every capture command.
- Update this document with new expected PNG names and artifact bundle contents.
- Keep generated PNGs out of Git. Use workflow artifacts for distribution.
- Do not treat generated artifacts as visual approval; human review still decides whether visual work is acceptable.
