# TitanCraft Visual Identity

## Purpose

This is the text-only Art Taste Pack for future Blender Asset Forge work. It defines what TitanCraft art must read as before agents create real assets. It is reference-only guidance; it does not authorize production art, Stage A art, gameplay code, or scene changes.

## Core Direction

TitanCraft art direction is **Polygonal Salvage Sci-Fi**:

- low-to-medium polygon construction;
- strong silhouettes readable in two seconds;
- simplified PBR materials with clear roughness, edge wear, and material separation;
- dramatic but readable lighting that supports routes, objectives, and hazards;
- visible wear, repairs, salvage, replacement panels, and practical field modifications;
- selective glow only for state, threat, interaction, or alien energy.

TitanCraft is **not**:

- cartoonish;
- photorealistic;
- block-based;
- voxel-like;
- glossy toy sci-fi;
- generic smooth-spacecraft art.

## Readability Hierarchy

1. Silhouette and scale must read before surface detail.
2. Large masses must explain purpose: shelter, crash, threat, tool, beacon, pickup, or path boundary.
3. Medium forms must explain construction: panels, ribs, bolts, cables, brackets, vents, repairs.
4. Small details must support use and damage, not hide an incorrect silhouette.
5. Glow must be sparse enough that every glow source has meaning.

## Crash Site MVP Fit

Visual work must stay inside the Crash Site MVP: a grounded survival FPS crash environment with one Galaxabrain enemy type, salvage resources, a workbench, save point, beacon, and first mechanical arm progression. The art layer must not imply multiplayer, procedural worlds, vehicles, a complete rocket, multiple enemy families, or non-MVP systems.

## Review Standard

A future asset is not art-approved because it validates technically. It must pass neutral-grey silhouette review, material-pass review, and contextual MVP fit review. Passing scripts or import checks only proves pipeline mechanics.
