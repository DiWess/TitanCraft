# Stage A Visual Target

## Scope

This file defines the visual target for future Stage A work. It does not create Stage A art and does not authorize scene edits.

## Target Read

Stage A should read as a small Crash Site slice with Polygonal Salvage Sci-Fi taste:

- heavy human wreckage with practical rectangular construction;
- volcanic impact basin terrain that avoids platform, slab, or board-game reads;
- one clear route communicated through massing, ash, value, wreckage, and negative space;
- foreground, midground, and background depth;
- muted off-white, graphite, basalt, ash, and orange accent palette;
- selective glow only for interaction, beacon state, save state, or alien threat.

## Stage A Must Avoid

- production asset claims without source, license, hash, and visual review evidence;
- route slabs, cards, ribbons, and artificial strips;
- toy hulls or smooth elegant spacecraft;
- generated meshes treated as final art without scorecard review;
- debug colors presented as a material pass;
- broad gameplay scope expansion.

## Approval Evidence Required Later

A future Stage A art claim needs PNG evidence, neutral-grey silhouette review, material-pass review, Godot import evidence, and explicit separation between technical validation and art approval.
