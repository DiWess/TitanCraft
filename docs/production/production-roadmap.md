# Production Roadmap

## Phase 0: Governance and pipeline complete

- Keep `README.md`, `AGENTS.md`, Agent Studio, Blender Asset Forge, Art Taste Pack, Visual Artifact Factory, and production-operation docs aligned.
- Maintain fail-closed evidence gates.

## Phase 1: MVP gameplay stability

- Stabilize the Crash Site loop.
- Fix repo-owned gameplay bugs without expanding the MVP.
- Validate player, pickups, workbench, mechanical arm, Scout, component, save point, beacon, victory, and defeat flow.

## Phase 2: Asset Forge production candidates

- Build standalone asset candidates through Blender Asset Forge.
- Export GLBs, generate manifests, and create review PNGs.
- Keep candidates out of production scenes until approved.

## Phase 3: Stage A visual replacement only after asset approval

- Replace placeholder visuals only after standalone asset approval and Stage A gate clearance.
- Require Visual Artifact Factory bundles and visual diagnosis.

## Phase 4: Full MVP polish

- Polish gameplay readability, UI, audio placeholders, collisions, and performance only within the locked MVP.
- Preserve the simple loop and Windows-first offline target.

## Phase 5: Windows playtest/export

- Produce a Windows export candidate.
- Validate offline launch, input, save/load, victory/defeat, and crash-free playtest evidence.

## Phase 6: Public demo preparation

- Prepare demo materials only after visual approval, gameplay validation, export proof, and release gates pass.
- Do not use marketing screenshots before approved visuals exist.

## Phase 7: Extended production

- Extend Crash Site MVP with composition guidance, balance validation, audio implementation, polish optimization, and platform testing.
- Implement Art Director composition guide, gameplay balance playtesting, audio & sound design, polish & optimization, and Windows platform expansion.
- All Phase 7 work operates within locked Crash Site scope; forbidden MVP features remain forbidden.
- See [`docs/production/phase-7-planning.md`](phase-7-planning.md) for detailed scope, gates, and success criteria.
