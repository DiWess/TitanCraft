## 1. BLOCKER found and fixed: legacy robot proxy rendered over the crafted first-person arm

The new crafted-arm capture (this slice) exposed a real production visual bug: crafting the
mechanical arm made `Player/Head/Camera3D/MechanicalArmVisual` visible, and that node still
carried the legacy third-party `Quaternius George.obj` ArrayMesh — so the player saw a full
humanoid robot filling half the first-person frame, hiding the approved MVP arm model that is
a child of the same node. The runtime path (`FirstPersonController.UpdateMechanicalArmVisual`)
toggles exactly this node, so every crafted playthrough had this defect. Evidence: first
capture iteration in `artifacts/visual-review/crafted-arm-first-person/` prior to the fix
(robot visible), post-fix captures (segmented arm only).

**Fix (minimal):** removed the `mesh` assignment from `MechanicalArmVisual` (node retained as
the script-controlled visibility group; integration tests assert node + visibility only, and
still pass), removed the dead `George.obj` ext_resource from `Main.tscn`, and re-tuned the
node transform so the MVP arm frames as a proper first-person viewmodel (lower-right forearm
pointing to screen center) instead of the legacy proxy's placement. The Quaternius asset file
itself stays in `assets/ThirdParty/` (licence-documented, now unreferenced by scenes).
