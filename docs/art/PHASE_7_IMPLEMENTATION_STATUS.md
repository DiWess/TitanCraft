## How to Use Phase 7 Composition Scene

### In Godot Editor

1. **Open Scene:**
   ```
   scenes/World/Phase7_Composition.tscn
   ```

2. **Run Scene (F5):**
   - Godot will instantiate all assets automatically
   - Console will log placement status (✓ markers)
   - Player spawns at (0, 0, 1.7) with FPS camera active

3. **Navigate (WASD):**
   - W/A/S/D to move forward/left/back/right
   - Mouse to look around
   - Arrow keys (if configured) for alternate movement

4. **Visual Validation:**
   - Confirm terrain is base layer (not floating)
   - Hull visible on far side (50m+ distance)
   - Workbench central and accessible
   - Beacon distant objective (long-distance silhouette)
   - Pickups clustered in three zones
   - Scout in arena zone (not spawning with player)
   - Mechanical arm visible in lower-right corner (FPS POV)

### Asset Path Configuration

All asset imports use relative paths from project root:
```
res://assets/Production/Generated/MVP_Pack_V1/TC_*.glb
```

When GitHub Actions generates GLB files, they will be placed at:
```
assets/Production/Generated/MVP_Pack_V1/
```

The Phase7CompositionManager will automatically discover and load them.

### Godot Version Compatibility

- **Godot Version:** 4.7 (specified in project.godot)
- **C# Version:** .NET 6+ (Godot 4.7 C# support)
- **Partial Class:** Required for Godot C# scripts (GD0001 compliance)

### Performance Considerations

- **Poly Count:** ~6,048 interactable polys + terrain (~10k) + hull (~1k) = ~17k total
- **Draw Distance:** Camera far clip set to 1000m (covers 50m+ to beacon)
- **Ambient Lighting:** 0.8x energy with 0.2x sky contribution (avoids over-bright scenes)
- **Shadow Rendering:** Enabled on directional light with split cascade
